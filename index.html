JavaScript Objects

Object Introduction

Objects are one of the fundamental data types in JavaScript. An object is a collection of key-value pairs where keys are strings (or Symbols) and values can be any data type. Objects allow you to store related data and functionality together, making them essential for organizing complex code.

// Creating an object using object literal notation
const person = {
    name: "John Doe",
    age: 30,
    city: "New York",
    isStudent: false
};

console.log(person); // { name: "John Doe", age: 30, city: "New York", isStudent: false }

// Objects can contain different data types
const user = {
    username: "john123",
    email: "john@example.com",
    age: 25,
    hobbies: ["reading", "gaming", "coding"],
    address: {
        street: "123 Main St",
        city: "Boston"
    }
};

Dot Notation vs. Bracket Notation

There are two ways to access object properties: dot notation and bracket notation. Dot notation is cleaner and more commonly used, but bracket notation is necessary when property names contain spaces, start with numbers, or are stored in variables.

const student = {
    firstName: "Sarah",
    lastName: "Johnson",
    age: 22,
    "favorite color": "blue",
    "grade-level": "senior"
};

// Dot Notation - Clean and simple
console.log(student.firstName);  // "Sarah"
console.log(student.age);        // 22

// Bracket Notation - Required for special property names
console.log(student["favorite color"]);  // "blue"
console.log(student["grade-level"]);     // "senior"

// Bracket notation with variables
const propertyName = "lastName";
console.log(student[propertyName]);  // "Johnson"

// Adding/Updating properties
student.email = "sarah@example.com";  // Add new property with dot notation
student["phone"] = "555-1234";        // Add new property with bracket notation

// Updating existing properties
student.age = 23;
student["firstName"] = "Sarah Marie";

console.log(student);

Nested Objects and Advanced Examples

Objects can contain other objects as properties, creating nested structures. This is useful for representing complex data like user profiles, company information, or configuration settings. You can access nested properties by chaining dot or bracket notation.

// Complex nested object example
const company = {
    name: "Tech Solutions Inc",
    founded: 2010,
    address: {
        street: "456 Innovation Drive",
        city: "San Francisco",
        state: "CA",
        zipCode: "94102",
        coordinates: {
            latitude: 37.7749,
            longitude: -122.4194
        }
    },
    employees: [
        {
            id: 1,
            name: "Alice Smith",
            position: "CEO",
            department: "Executive",
            salary: 150000,
            contact: {
                email: "alice@techsolutions.com",
                phone: "555-0001"
            }
        },
        {
            id: 2,
            name: "Bob Jones",
            position: "Lead Developer",
            department: "Engineering",
            salary: 120000,
            contact: {
                email: "bob@techsolutions.com",
                phone: "555-0002"
            }
        }
    ],
    departments: {
        engineering: {
            name: "Engineering",
            headCount: 25,
            budget: 2000000
        },
        sales: {
            name: "Sales",
            headCount: 15,
            budget: 1000000
        }
    }
};

// Accessing nested properties
console.log(company.address.city);  // "San Francisco"
console.log(company.address.coordinates.latitude);  // 37.7749
console.log(company.employees[0].name);  // "Alice Smith"
console.log(company.employees[1].contact.email);  // "bob@techsolutions.com"
console.log(company.departments.engineering.headCount);  // 25

// Using bracket notation for nested access
console.log(company["employees"][0]["position"]);  // "CEO"

Create Object with New Keyword

The new Object() syntax creates an empty object using JavaScript's built-in Object constructor. While this method works, object literal notation is generally preferred for its simplicity. However, understanding this approach is important for working with constructor functions and classes.

// Creating object with new Object()
const car = new Object();
car.brand = "Toyota";
car.model = "Camry";
car.year = 2023;
car.color = "Silver";
car.start = function() {
    console.log("Engine started!");
};

console.log(car);
car.start();  // "Engine started!"

// Constructor Function Pattern
function Person(name, age, occupation) {
    this.name = name;
    this.age = age;
    this.occupation = occupation;
    this.introduce = function() {
        console.log(`Hi, I'm ${this.name}, ${this.age} years old, and I work as a ${this.occupation}`);
    };
}

// Creating instances with new keyword
const person1 = new Person("Emily", 28, "Designer");
const person2 = new Person("Michael", 35, "Engineer");

person1.introduce();  // "Hi, I'm Emily, 28 years old, and I work as a Designer"
person2.introduce();  // "Hi, I'm Michael, 35 years old, and I work as a Engineer"

console.log(person1.name);  // "Emily"
console.log(person2.occupation);  // "Engineer"

// ES6 Class Syntax (Modern Approach)
class Animal {
    constructor(name, species, age) {
        this.name = name;
        this.species = species;
        this.age = age;
    }
    
    makeSound() {
        console.log(`${this.name} makes a sound`);
    }
}

const dog = new Animal("Buddy", "Dog", 3);
dog.makeSound();  // "Buddy makes a sound"

This Keyword

The this keyword refers to the object that is executing the current function. In object methods, this refers to the object itself. Understanding this is crucial for working with object-oriented JavaScript and avoiding common pitfalls.

// 'this' in object methods
const calculator = {
    value: 0,
    add: function(num) {
        this.value += num;
        return this;  // Return this for method chaining
    },
    subtract: function(num) {
        this.value -= num;
        return this;
    },
    multiply: function(num) {
        this.value *= num;
        return this;
    },
    getResult: function() {
        return this.value;
    }
};

// Method chaining using 'this'
calculator.add(10).multiply(2).subtract(5);
console.log(calculator.getResult());  // 15

// 'this' in different contexts
const user = {
    name: "David",
    age: 30,
    greet: function() {
        console.log(`Hello, my name is ${this.name}`);
    },
    getBirthYear: function() {
        return new Date().getFullYear() - this.age;
    },
    hobbies: ["reading", "gaming"],
    showHobbies: function() {
        console.log(`${this.name}'s hobbies:`);
        this.hobbies.forEach(function(hobby) {
            // 'this' is undefined here in regular function
            console.log(hobby);
        });
    },
    showHobbiesArrow: function() {
        console.log(`${this.name}'s hobbies:`);
        // Arrow functions inherit 'this' from parent scope
        this.hobbies.forEach((hobby) => {
            console.log(`${this.name} likes ${hobby}`);
        });
    }
};

user.greet();  // "Hello, my name is David"
console.log(user.getBirthYear());  // 1995 (if current year is 2025)
user.showHobbiesArrow();

Create Object with Create Method

The Object.create() method creates a new object using an existing object as the prototype. This allows for prototypal inheritance, where new objects can inherit properties and methods from a parent object without using constructor functions or classes.

// Basic Object.create() usage
const personPrototype = {
    greet: function() {
        console.log(`Hello, I'm ${this.name}`);
    },
    getInfo: function() {
        return `${this.name} is ${this.age} years old`;
    }
};

// Create new object with personPrototype as prototype
const john = Object.create(personPrototype);
john.name = "John";
john.age = 25;

john.greet();  // "Hello, I'm John"
console.log(john.getInfo());  // "John is 25 years old"

// Create object with properties
const jane = Object.create(personPrototype, {
    name: {
        value: "Jane",
        writable: true,
        enumerable: true
    },
    age: {
        value: 30,
        writable: true,
        enumerable: true
    },
    occupation: {
        value: "Developer",
        writable: true,
        enumerable: true
    }
});

jane.greet();  // "Hello, I'm Jane"

// Prototypal inheritance example
const vehicle = {
    type: "Vehicle",
    start: function() {
        console.log(`${this.brand} ${this.model} is starting...`);
    },
    stop: function() {
        console.log(`${this.brand} ${this.model} has stopped.`);
    }
};

const car = Object.create(vehicle);
car.brand = "Honda";
car.model = "Civic";
car.wheels = 4;

car.start();  // "Honda Civic is starting..."

// Check prototype chain
console.log(Object.getPrototypeOf(car) === vehicle);  // true

Create Object with Assign Method

The Object.assign() method copies properties from one or more source objects to a target object. This is commonly used for merging objects, cloning objects, or adding properties to existing objects. It's particularly useful for creating new objects based on existing ones.

// Basic Object.assign() - Copying properties
const target = { a: 1, b: 2 };
const source = { b: 3, c: 4 };

const result = Object.assign(target, source);
console.log(result);  // { a: 1, b: 3, c: 4 }
console.log(target);  // { a: 1, b: 3, c: 4 } - target is modified!

// Creating new object (cloning)
const original = {
    name: "Original",
    value: 100
};

const clone = Object.assign({}, original);
console.log(clone);  // { name: "Original", value: 100 }

// Merging multiple objects
const defaults = {
    theme: "light",
    language: "en",
    notifications: true
};

const userSettings = {
    theme: "dark",
    fontSize: 14
};

const finalSettings = Object.assign({}, defaults, userSettings);
console.log(finalSettings);
// { theme: "dark", language: "en", notifications: true, fontSize: 14 }

// Practical example: Creating user profile
const baseUser = {
    role: "user",
    active: true,
    createdAt: new Date()
};

const userData = {
    name: "Alex Johnson",
    email: "alex@example.com",
    age: 28
};

const permissions = {
    canEdit: false,
    canDelete: false,
    canView: true
};

const fullUserProfile = Object.assign({}, baseUser, userData, permissions);
console.log(fullUserProfile);

// Note: Object.assign() performs shallow copy
const obj1 = {
    a: 1,
    nested: { b: 2 }
};

const obj2 = Object.assign({}, obj1);
obj2.nested.b = 999;
console.log(obj1.nested.b);  // 999 - nested object is shared!

// Alternative: Spread operator (ES6+)
const obj3 = { x: 1, y: 2 };
const obj4 = { y: 3, z: 4 };
const merged = { ...obj3, ...obj4 };
console.log(merged);  // { x: 1, y: 3, z: 4 }
